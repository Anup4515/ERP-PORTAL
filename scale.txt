  next.config.ts            | Standalone output, hardcoded ngrok dev origin

Auth Flow
---------
  Login
    -> credentials provider
    -> users table (role_id 4=admin, 5=teacher)
    -> bcrypt verify
    -> resolveSchoolId()
    -> JWT {user_id, school_id, role}
    -> middleware checks token on every request
    -> API routes call getAuthContext(["allowed_roles"])
    -> queries scoped by partnerUserId (WHERE partner_id = ?)

Multi-Tenancy (Current)
-----------------------
  - All 49 protected API routes scope data by partner_id / partnerUserId
  - Auth context resolves tenant at request time
  - No cross-tenant data leakage found

Route Inventory

  - 12 school-admin pages, 9 teacher pages, auth pages, setup page
  - All page components are client-side ("use client")

4.2 Frontend
------------
  Finding                                             | Impact
  ----------------------------------------------------|-------------------------
  All pages are client-side ("use client")             | No SSR/ISR benefits
  No dynamic imports for heavy libs (PDF, Excel)       | Larger bundle
  No next/font optimization                            | Font loading not optimized
  428+ hardcoded strings (no i18n)                     | Can't localize
  Good loading/skeleton states                         | Good UX
  next/image used correctly                            | Good

4.3 Scalability Limits
----------------------
  Threshold                 | Bottleneck                    | Fix
  --------------------------|-------------------------------|----------------------------
  >100 concurrent users     | No caching, all DB hits       | Add Redis
  >500 concurrent users     | 50-connection DB pool         | Increase pool, read replicas
  >1000 concurrent users    | In-memory rate limiter        | Distributed rate limiter
  Multi-server deployment   | Rate limiter state lost       | Redis-backed rate limiter
  Large CSV imports         | Entire file in memory         | Streaming parser + size limit
